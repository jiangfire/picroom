# Picroom Code Review — 2025-07

> **Date**: 2026-07-06
> **Scope**: Full codebase audit (102 `.rs` files, 112 tests)
> **Verdict**: Infrastructure layer (SigV4, RBAC, AVIF, Argon2, SQLite queue) is
> solid and well-tested. **Business wiring is almost entirely broken** — the
> binary runs in dev mode, the router mounts no auth layer, and DB integration
> is severed in three places. Should not be exposed to the public internet in
> its current form.

---

## 1. Critical — Blocks v1 Release

### 1.1 Authentication Chain Severed

| Component | Implementation | Wiring |
|---|---|---|
| `JwtService` (issue + verify) | ✅ Complete, tested | 🔴 Not connected to any HTTP path |
| `PasswordHasher` (Argon2id) | ✅ Complete, tested | 🔴 `auth::login` handler returns 501 |
| `HttpOidcClient` (discover + exchange + userinfo) | ✅ Complete, 6 wiremock tests | 🔴 OIDC routes not registered in router |
| `AuthUser` extractor | ❌ Always returns `Err(unauthorized)` | 🔴 No handler uses it |
| `require_auth` middleware | ❌ Pass-through `Ok(next.run(req))` | 🔴 Never attached to router |
| `dev_user` | Hardcoded `UserId(Uuid::nil())` | 🔴 All uploads attributed to nil UUID |

### 1.2 S3 Compatibility Layer — All 501

- `object.rs`: PUT/GET/HEAD/DELETE → `StatusCode::NOT_IMPLEMENTED`
- `multipart.rs`: init/part/complete/abort → 501
- `sigv4.rs`: Algorithm verified against AWS test vectors, but **no handler calls `verify()`**
- S3 errors returned as JSON; AWS clients expect XML
- `ListObjectsV2` route wrongly points to `head_object`

### 1.3 Upload Does Not Persist to DB + Variants Not Recorded

| Data flow | Status |
|---|---|
| `UploadService.upload()` → INSERT `images` | 🔴 Not written |
| API handler → `repo.insert()` | 🔴 Code exists but `image_repo=None` in dev AppState |
| Worker → INSERT `image_variants` | 🔴 Not written (no variant repository in `ProcessorDeps`) |
| `GET /images/:id` → SELECT | 🔴 Returns 500 "image repo not configured" |
| `QuotaService` | 🔴 Stub: `remaining_user` → `u64::MAX`, `charge_user` → `Ok(())` |

### 1.4 MinIO Configuration Ignored

`docker-compose.yml` configures full MinIO env vars, but `api_cmd.rs` and
`worker_cmd.rs` **hardcode `LocalDriver("data/")`**, ignoring
`PICROOM_STORAGE__POLICIES__MINIO__*`.

### 1.5 Security Issues

| Issue | Severity | Location |
|---|---|---|
| DB URL (with password) leaked to stderr | Critical | `api_cmd.rs:24` |
| Default `jwt_secret="change-me"` with no validation | Critical | `config.rs:150` |
| No HTTP-layer body size limit (DoS) | Critical | Router missing `RequestBodyLimitLayer` |
| IDOR: image GET/DELETE/list has no owner check | Critical | `images.rs` handlers |
| All endpoints zero authentication | Critical | Router mounts no auth layer |
| SigV4 signature comparison not constant-time | High | `sigv4.rs:189` `eq_ignore_ascii_case` |
| `S3Error::SignatureMismatch` leaks expected signature | High | `error.rs:16-22` |

## 2. Major — Spec Violations / Missing Features

### 2.1 API Endpoint Coverage (19 spec endpoints)

| Status | Count | Endpoints |
|---|---|---|
| ✅ Working | 5 | `POST /images`, `GET /images`, `GET /images/:id`, `DELETE /images/:id`, `GET /healthz` |
| 🔴 501 stub | 6 | `POST /auth/login`, `POST /teams`, `GET /teams/:id`, `POST /teams/:id/members`, `GET /audit`, `POST /admin/users` |
| ⚫ Missing | 8 | `GET /me`, `GET /images/:id/{file,avif,webp,thumbnail}`, `PATCH /admin/users/:id/role`, `GET+POST /admin/storage/policies`, OIDC routes |

### 2.2 Imaging Processor Trait — Placeholder Implementations

`crates/imaging/src/processor/{avif,webp,thumbnail,watermark}.rs` return input
bytes unchanged. The **worker** (`crates/worker/src/processor.rs`) has real
encoders, so the async pipeline produces correct output — but the synchronous
`Pipeline` API is a no-op.

### 2.3 spec §5.3 Violation: `unwrap()` in Non-Test Code

| File | Lines | Context |
|---|---|---|
| `infra/src/cache.rs` | 47, 59, 65 | `InMemoryCache` lock().unwrap() |
| `audit/src/sink.rs` | 48, 55 | `InMemoryAuditSink` |
| `worker/src/dlq.rs` | 42, 49 | `InMemoryDlq` |
| `domain/src/clock.rs` | 37, 50 | `FakeClock` (no `#[cfg(test)]`) |
| `worker/src/processor.rs` | 118 | `StorageKey::parse(...).expect(...)` — panics on bad input |

### 2.4 Worker Retry Has No Backoff Delay

`RetryPolicy::delay_secs()` computes exponential backoff but `pool.rs::run_into`
never calls `sleep(delay)` — failed jobs are retried instantly 5 times, then
DLQ'd.

### 2.5 Docker Compose Integration Broken

- No migrate step in compose — tables don't exist on startup
- MinIO env vars ignored (see §1.4)
- `NoopAuditSink` discards all audit events
- `image_repo=None` makes list/get/delete return 500
