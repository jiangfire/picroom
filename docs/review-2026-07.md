# Picroom Code Review — 2026-07

> **Date**: 2026-07-08
> **Baseline**: `docs/review-2025-07.md` (2026-07-06)
> **Scope**: Full re-audit. Verify each prior finding against HEAD; surface
> regressions introduced since.
> **Delta since baseline**: 19 commits, all `ci:` / `style:` — no business-logic
> commit touched the auth, S3, worker, or persistence paths.
> **Verdict**: The Postgres happy path (upload → DB → worker → variants → audit)
> is now wired and works. **However security is worse than at baseline**: login
> issues a JWT without verifying the password, and the issued token cannot pass
> the extractor anyway (sub mismatch + empty scopes). The CI gates that the spec
> treats as acceptance criteria (S5 coverage, S8 audit, S9 deny) have been
> **relaxed to make the pipeline green**, not made actually clean. The worker
> retry-backoff claim in ADR-0007 is contradicted by the source. Not safe to
> expose to the public internet.

## 3. Unfixed Since Baseline (carried forward)

### 3.1 Authentication chain (Critical)

- OIDC routes are unmounted stubs returning `501`: `crates/api/src/handlers/auth.rs:55-63`; no `oidc` entry in `crates/api/src/router.rs:18-40`. → Spec S13 fails.
- `dev_user` nil in production (see R6).

### 3.2 S3 compatibility (Critical)

- `sigv4::verify` / `parse_authz` are **dead code** — only called from unit tests inside `sigv4.rs`. No handler invokes them. The S3 router is mounted open: `crates/api/src/router.rs:42` with a misleading comment. → Spec S11/S12 fail.
- Signature comparison is not constant-time: `crates/s3compat/src/sigv4.rs:194` `expected.eq_ignore_ascii_case(...)`. No `subtle`/`ConstantTimeEq` in the tree.
- `S3Error::SignatureMismatch` leaks the expected signature in its `Display`, which reaches the response body at `error.rs:43`: `error.rs:16-22`. Combined with the non-constant-time compare this is a signature oracle (latent until §3.2 verify is wired).
- `S3Error` still serializes as JSON, not XML (`error.rs:41-44`). Handlers work around it with private `s3_xml_error` helpers (`object.rs:101-106`, `list.rs:76-80`); the type itself is non-conformant.
- Multipart silent data loss (see R7).

### 3.3 Worker robustness (Major)

- **No retry backoff.** `RetryPolicy::delay_secs` exists and is unit-tested but is **never called outside `retry.rs` tests**. `pool.rs::run_into` (`crates/worker/src/pool.rs:59-71`) fails a job straight back to `pending`; `PgJobQueue::fail` (`db_queue.rs:107-114`) flips it to `pending` immediately → a failing job is retried 5× back-to-back, then DLQ'd. **ADR-0007 §"Worker retry applies `RetryPolicy::delay_secs()` as `sleep()`" is false** (`docs/adr/0007-security-hardening.md:72`).
- `JobKind::ApplyWatermark` / `StripExif` return `Err`: `crates/worker/src/processor.rs:69-70`.
- `QuotaService` is a stub (`remaining_user → u64::MAX`, `charge_user → Ok(())`): `crates/service/src/quota.rs:17-29`.
- `DeleteService` is a stub (writes only an audit event; does not delete storage or DB row); the DELETE handler bypasses it entirely: `crates/service/src/delete.rs:25-44`.

### 3.4 Docker bootstrap (Major)

- `docker/docker-compose.yml` runs `api`/`worker` with **no migration step**. On a fresh `compose up` the `images`, `image_variants`, `audit_events`, `jobs` tables do not exist → first upload 500s. No init container, no entrypoint migration. → Spec S10 fails.

### 3.5 `unwrap()` / panic risk in non-test code (spec §5.3 violation)

- `crates/infra/src/cache.rs:47,59,65` — `RwLock` `.unwrap()` ships in release.
- `crates/worker/src/processor.rs:153-158` — new `panic!` fallback when a generated key fails to parse.
- `crates/audit/src/sink.rs:48,57` and `crates/worker/src/dlq.rs:42,49` — converted from `.unwrap()` to `.expect("mutex poisoned")` (still panics on poison).

### 3.6 Observability (Minor→Major)

- `/metrics` returns a static placeholder string: `crates/api/src/handlers/system.rs:57-62`.
- `/readyz` DB check is hardcoded `true`: `crates/api/src/handlers/system.rs:19-26` ("Full implementation would ping the database pool."). Storage is probed.

### 3.7 Error responses leak internals (Major)

- `crates/api/src/error.rs:92,94` funnels unmatched `ServiceError`/`DomainError::Internal` into the response body.
- `crates/storage/src/driver/s3.rs:364-368,391-395,419-423,519-520` embed raw S3 response bodies; `local.rs:72-75,87-90,125,140,144,153-156` embed `path.display()`. These reach clients via `ServiceError::Storage(#[from])` → `error.rs:74`.

### 3.8 Admin CLI (Minor)

- `admin migrate revert/status` do not exist (only `migrate_run`): `crates/admin/src/migrate.rs:15`.
- `admin audit tail` returns `Ok(Vec::new())`: `crates/admin/src/audit_cmd.rs:15-20`.

### 3.9 Quality / spec-conformance (Minor)

- **No MIT/SPDX headers** in any of the ~102 `.rs` files. → Spec S15 fails (0/102).
- Dead code: `crates/service/src/query.rs:75,80` (`_stub`, `_bytes`).
- `utoipa`/`utoipa-swagger-ui` are declared (`Cargo.toml:97-98`, `crates/api/Cargo.toml:35`) but never imported — `docs/api/openapi.yaml` is hand-written.
- `jwt_secret` default `"change-me"` is still accepted by the config layer (`crates/infra/src/config.rs:150`); the only guard is a release-mode `assert!` in `api_cmd.rs:56` that does not cover debug builds or the worker binary.
- `benches/` is empty (S3/S4 unverifiable); workspace `tests/` contains no E2E harness.
- `docs/deployment.md`, `docs/operations.md`, `docs/security.md` do not exist.

## 5. Recommended Fix Priority

### P0 — Make the system usable and honest

1. **P0-1** Login verifies password via `PasswordHasher::verify`; issue JWT with `sub = user_id` and `scopes = [role]` (fixes R1, R2, R3). Requires a `find_user_by_email` returning id+role+hash.
2. **P0-2** `require_auth` verifies the token; image GET/DELETE/list use `AuthUser` (non-optional) — closes R4, R5.
3. **P0-3** Remove `dev_user = Uuid::nil()` from production; take actor from `AuthUser` (R6).
4. **P0-4** `docker-compose.yml` migration init container + `depends_on` (§3.4).
5. **P0-5** Worker calls `RetryPolicy::delay_secs()` + `sleep`; correct ADR-0007 (§3.3).

### P1 — Restore CI honesty

6. **P1-6** Remove `cargo audit --ignore`; resolve rsa/tokio-tar (upgrade or exclude testcontainers from audit).
7. **P1-7** `deny.toml` back to MIT-only (document any waiver explicitly); `wildcards = "deny"`.
8. **P1-8** Pin the 16 major-only deps to minor.
9. **P1-9** `tarpaulin --fail-under 80` and add `coverage` to the `required` gate.

### P2 — S3 real authentication

10. **P2-10** Wire `sigv4::verify` into the S3 path; constant-time compare (`subtle`); remove the expected-signature leak.
11. **P2-11** Multipart: revert to honest `501` or implement real assembly (end silent data loss, R7).

### P3 — Quality

12. **P3-12** Eliminate non-test `unwrap`/`expect`/`panic!` (cache, sink, dlq, processor).
13. **P3-13** Real `/metrics` (Prometheus) and `/readyz` DB ping.
14. **P3-14** Sanitize error responses (no S3 bodies / FS paths to clients).
15. **P3-15** `admin migrate revert/status` + `audit tail`; honest `501` for Watermark/StripExif.
16. **P3-16** Implement or formally defer QuotaService / DeleteService.
17. **P3-17** Validate `jwt_secret` in the config layer.
18. **P3-18** Delete dead code (`_stub`/`_bytes`, unused `utoipa`).
19. **P3-19** MIT/SPDX headers across all source files (S15).
20. **P3-20** Write `docs/deployment.md`, `docs/operations.md`, `docs/security.md`.
